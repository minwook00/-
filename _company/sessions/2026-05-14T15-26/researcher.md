# 관계 실패 사례 데이터 수집 및 구조화 계획 (CEO 지시사항 기반)

## 🎯 목표
웹사이트 및 커뮤니티에서 '가치관 충돌'에 기반한 관계 실패 사례(S-A-A 구조) 최소 50개 확보 및 M1~M4 가치관 불일치 데이터 수집.

## 🛠️ 방법론
1. **검색 키워드 활용:** 이전 단계에서 도출된 Tier 2/3 키워드를 기반으로 웹사이트와 커뮤니티를 검색하여 잠재적 사례를 식별한다.
2. **S-A-A 구조화 적용:** 수집된 각 사례를 다음 구조에 맞추어 분석을 준비한다.
    *   **S (Situation):** 구체적인 상황 설명 (고민글 내용)
    *   **A (Action):** 상대방의 행동 및 나의 반응 (실제 대화 내용)
    *   **A (Attitude/Outcome):** 결과와 감정적 충돌 지점
3. **M1~M4 지표 초기 데이터 수집:** 각 사례에서 핵심 가치관 불일치를 정량화할 수 있는 초기 데이터를 추출한다.
    *   **M1 (상호 존중):** 상대방이 나의 의견을 얼마나 수용했는지?
    *   **M2 (자율성/독립성):** 관계 내에서 각자의 독립적인 영역을 인정받았는지?
    *   **M3 (안정성/보장):** 미래에 대한 예측 가능성과 안정감을 제공받았는지?
    *   **M4 (공유/헌신):** 감정, 시간, 자원을 얼마나 공유했는지?

## 📊 데이터 구조 설계 (DB Schema 초안)
| 필드명 | 유형 | 설명 | 측정 기준 (예시) |
| :--- | :--- | :--- | :--- |
| `case_id` | INT | 고유 식별자 | - |
| `source` | TEXT | 출처 (웹사이트/커뮤니티 구분) | - |
| `situation` | TEXT | 상황 요약 (S) | 키워드 기반 요약 |
| `action` | TEXT | 핵심 행동 및 대화 내용 (A) | 인용 또는 요약 |
| `outcome` | TEXT | 결과 및 감정적 충돌 (A) | 관계 실패의 원인 명시 |
| `m1_score` | INT (1-5) | 상호 존중 점수 | - |
| `m2_score` | INT (1-5) | 자율성 인정 점수 | - |
| `m3_score` | INT (1-5) | 안정성/보장 점수 | - |
| `m4_score` | INT (1-5) | 공유 및 헌신 정도 | - |
| `core_conflict` | TEXT | 핵심 가치관 충돌 요약 | M1~M4 기반 추론 |

## 🚀 실행 계획 (다음 단계)
1. **데이터 수집:** 정의된 키워드를 사용하여 웹사이트와 커뮤니티에서 잠재적 사례를 검색하고 수집한다.
2. **S-A-A 구조화:** 수집된 각 사례에 대해 S-A-A 프레임워크를 적용하여 뼈대를 구축한다.
3. **M1~M4 측정:** 전문가의 판단을 통해 초기 M1~M4 점수를 할당하고 데이터베이스로 구조화한다.
