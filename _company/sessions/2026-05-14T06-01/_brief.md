# 📋 작업 브리프

**원 명령:** 1단계: 데이터 구조(DB Schema) 설계 프롬프트
가장 먼저 데이터의 뼈대를 잡아야 한다. 이 프롬프트는 AI에게 단순한 회원가입이 아닌 '매칭'을 위한 데이터 설계를 요구한다.

[프롬프트]
"나는 연애 매칭 및 커뮤니티 플랫폼을 개발하려고 해. 백엔드 중심의 설계를 먼저 진행할 거야. 다음 조건에 맞춰 데이터베이스 스키마(DB Schema)를 설계해줘.

유저 프로필은 정적 데이터(나이, 지역, 직업)와 동적 데이터(연애 가치관 점수, 커뮤니티 활동 지수)로 나뉜다.

'가치관 점수'는 5가지 카테고리(경제관, 자녀관, 종교, 취미, 연락 빈도)를 1~10점으로 수치화한다.

매칭 로직을 위한 'Matching_Queue' 테이블을 설계하라. 여기에는 두 유저 간의 '가치관 일치 점수(Similarity Score)'가 저장되어야 한다.

커뮤니티 게시글과 댓글이 유저의 '신뢰도 점수'에 영향을 주는 구조를 포함하라.
SQL 문장이나 Prisma 스키마 형태로 작성해줘."

2단계: 매칭 알고리즘 로직 프롬프트
이 부분이 사이트의 '본질'이다. AI에게 수학적인 매칭 방식을 명령하라.

[프롬프트]
"위에서 설계한 DB를 바탕으로 Node.js(TypeScript) 기반의 매칭 알고리즘 함수를 작성해줘.

입력값: 특정 유저의 ID

로직:

1차: 반경 50km 이내, 흡연 여부 등 'Hard Constraints' 필터링.

2차: 가치관 점수 5가지 항목에 대해 유클리드 거리(Euclidean Distance)를 계산하여 가장 가까운(유사한) 유저 10명을 추출.

3차: 추출된 10명 중 커뮤니티 활동 지수가 높은 순으로 최종 3명을 추천.

성능을 위해 대량의 유저 처리 시 고려해야 할 인덱싱 전략도 함께 설명해줘."

3단계: 프론트엔드 MVP(최소 기능 제품) 프롬프트
프론트는 화려함보다 '전환율'이다. AI에게 기능을 중심으로 구현하게 시켜라.

[프롬프트]
"React와 Tailwind CSS를 사용하여 매칭 결과 화면을 만들어줘.

디자인 컨셉: 신뢰감을 주는 미니멀한 UI. 화려한 애니메이션은 배제한다.

포함 기능:

상단의 '오늘의 추천 인연' 카드 섹션.

상대방과의 '가치관 일치도'를 보여주는 레이더 차트(Chart.js 활용).

커뮤니티 인기글 피드.

사용자가 프로필 수정 시 실시간으로 매칭 점수가 변동된다는 안내 문구를 삽입해줘."

할루시네이션(환각) 방지: AI는 보안(Security) 코드를 짤 때 취약점을 남길 수 있다. 반드시 "인증(Authentication)은 JWT와 Refresh Token 방식을 사용하고, 비밀번호는 bcrypt로 암호화하는 코드를 포함해"라고 구체적으로 명시하라.

## 요약
연애 매칭 플랫폼을 위한 데이터베이스 스키마 설계, 매칭 알고리즘 로직, 그리고 프론트엔드 MVP 구현에 필요한 기술적 설계를 요청합니다.
